Smart Energy Monitoring — run order
====================================

1) Prerequisites: Node.js 18+, Python 3 with pip, Docker Desktop (for Mosquitto + MongoDB).

2) Start infrastructure:
   docker compose up -d
   (MongoDB on 27017, MQTT on 1883)

3) Python ML deps:
   pip install -r ml/requirements.txt

4) Backend:
   cd backend
   npm install
   npm start
   Copy .env.example to .env. SKIP_AUTH=true opens the dashboard without login.
   With SKIP_AUTH=false: use Sign up, or demo@example.com / demo123 (seeded on first run).
   Google Sign-In: set GOOGLE_CLIENT_ID in backend/.env and the same OAuth Web client ID in frontend/.env as VITE_GOOGLE_CLIENT_ID (Google Cloud Console → Authorized JavaScript origins: http://localhost:5173).

5) Device simulator (new terminal):
   cd simulator
   npm install
   npm start

6) Frontend (new terminal):
   cd frontend
   npm install
   npm run dev
   Open http://localhost:5173

7) Optional: production build
   cd frontend && npm run build && npm run preview

Notes
-----
- Chart data updates in real time via WebSocket (Socket.IO) when new MQTT readings are stored.
- Export CSV and browser Print-to-PDF from the dashboard toolbar.
- High usage alerts fire when a single reading exceeds ~3500 W (adjust in backend/src/index.js HIGH_WATTS).
